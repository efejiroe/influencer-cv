# Data Model Reference

This file serves as a lightweight schema reference for the `data/` directory to avoid re-reading large CSV files in future AI sessions.

## Core Tables

### 1. Polling Data (Fact Table)
**File:** `data/tracking_data.csv`
**Granularity:** Hourly/Daily polling per video.
**Schema:**
- `date_time`: POSIXct timestamp of the poll.
- `video_id`: YouTube Video ID (unique per video).
- `views`: Cumulative view count at poll time.
- `likes`: Cumulative like count.
- `comments`: Cumulative comment count.
- `sentiment_pos`: % Positive comments (Bing sentiment).
- `sentiment_neu`: % Neutral comments.
- `sentiment_neg`: % Negative comments.

### 2. Influencer Registry (Dimension Table)
**File:** `data/influencers.csv`
**Schema:**
- `Influencer`: Proper name of the creator.
- `Handle`: @handle (Instagram/YouTube).
- `Channel ID`: YouTube Channel ID.

### 3. Video Lookup
**File:** `data/channel-video-lookup.csv`
**Used for:** Mapping videos to their parent channels and launch dates.
- `video_id`
- `channel_id`
- `start_date`

### 4. Active Tracking State
**File:** `data/active_tracking.csv`
**Used for:** Managing the tracker process status.
- `video_id`
- `start_time`
- `channel_id`
- `sentiment_pulled`: Boolean (T/F) indicating if the 24h sentiment snapshot was taken.

---

## Power BI Export
**File:** `data/pbi_fact_polling.csv` (Generated by `pbi.R`)
**Format:** Flattened table joining polling metrics with influencer metadata for direct visualization.
